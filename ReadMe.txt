Прокси сервер на Linux для логгирования входящих к БД на PostgreSQL запросов.

Используемые библиотеки:
C++ STL
Linux:
    sys/epoll.h
    sys/socket.h
    netinet/in.h
    arpa/inet.h

Код готов к сборке с помощью Cmake:
- mkdir build
- cd build
- cmake ..
- cmake --build .

Перед запуском приложения необходимо:
1. Отключить соединение клиентов по SSL в файле конфигурации PostgreSQL 
    postgresql.conf: ssl on -> ssl off

2. Изменить порт для подключения к PostgreSQL в файле конфигурации
    postgresql.conf: port = 5432 -> port = 5433

Приложение может быть запущено из терминала для отладки или из папки (переместится в фоновый процесс).
Файл лога создается в каталоге приложения.

P.S.
В коде имеются комментарии и отдельный закомментированный блок кода для отладки.
В репозитарии имеется уже собранное приложение build_v1.0/pg_proxy
